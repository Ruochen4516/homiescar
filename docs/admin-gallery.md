# HomiesCar 相册管理员使用说明

网站相册采用静态生成方式，适合 GitHub Pages。

## 上传流程

1. 打开 `admin.html`。
2. 选择对应活动。
3. 拖入该活动的全部照片。
4. 在右侧“照片编辑”里调整照片：
   - 删除不需要的照片。
   - 点击“设为封面”，把照片放到第一张。
   - 使用“上移 / 下移”调整相册顺序。
   - 编辑照片说明。
   - 使用“清空本次选择”重置当前活动队列。
5. 点击“写入项目文件夹”。
6. 在浏览器弹出的目录选择器中选择 `D:\Codex\HomiesCar`。
7. 工具会写入：
   - `assets/galleries/<活动ID>/`
   - `gallery-data.js`
8. 本地确认无误后提交并推送到 GitHub Pages。

## 注意

- 一个活动可以上传多张照片，第一张会作为活动卡封面。
- 选择活动后，如果已有线上相册，右侧会加载已有照片；删除后重新写入，会从前台相册资料中移除。
- 静态网页无法自动删除 GitHub 仓库中已经发布过的旧图片文件；它只会更新前台读取的 `gallery-data.js`。
- 找不到照片的活动不会套用其他活动照片，前台显示“相册整理中”。
- 推荐使用 Chrome 或 Edge，因为它们支持直接写入本地文件夹。
- 如果浏览器不支持直接写入，可以使用“下载生成文件”，再手动放入对应目录。
